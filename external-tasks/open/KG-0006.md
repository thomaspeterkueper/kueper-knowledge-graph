## Ausgangslage

`kxf-learning-modules-0.1.json` enthält vier Schema-Inkonsistenzen, die sich bei
jedem KXF-Export, jeder SSF-Integration und jedem NOXIA-Unlock-Lookup reproduzieren.

---

## Punkt 1 — Subjektcode-Konflikte

Zwei konkurrierende Präfix-Sätze sind im Umlauf:

| Variante A | Variante B | Domäne |
|------------|------------|--------|
| `CHE`      | `CHM`      | Chemie |
| `EAR`      | `GEO`      | Geowissenschaften |
| —          | `TEC`      | Technik / Engineering |

### Vorgeschlagener Entscheid

Kanonische Domänenpräfixe:

| Präfix | Domäne |
|--------|--------|
| `PHY`  | Physik |
| `CHM`  | Chemie (IUPAC-konform; `CHE` war Anglizismus) |
| `MAT`  | Mathematik |
| `AST`  | Astronomie |
| `GEO`  | Geowissenschaften |
| `BIO`  | Biologie |
| `TEC`  | Technik / Engineering (neu) |
| `INF`  | Informatik |

Migration: `CHE-*` → `CHM-*`, `EAR-*` → `GEO-*`. `legacyId` behält alte Werte.

---

## Punkt 2 — snake_case vs. camelCase

Feldnamen mischen beide Konventionen. Ausreißer:
`prereq_ids`, `subject_code`, `depth_level`

### Vorgeschlagener Entscheid

Kanonisch: `camelCase` für alle KXF-JSON-Feldnamen.

| Alt | Neu |
|-----|-----|
| `prereq_ids` | `prereqIds` |
| `subject_code` | `subjectCode` |
| `depth_level` | `depthLevel` |

---

## Punkt 3 — Unlock-Semantik

Drei verschiedene Strukturen im Umlauf: `noxiaKeys`, `unlocks`, `keys`.

### Vorgeschlagener Entscheid

Kanonisch: `noxiaGrants` mit optionalem `condition`-Feld.

```json
"noxiaGrants": [
  { "key": "SENSOR:SPECTRAL", "condition": "completed" }
]
```

Kurzform (condition = completed): `"noxiaGrants": ["SENSOR:SPECTRAL"]`

---

## Punkt 4 — Unaufgelöste Präfixe KD / CMP

- `KD` (Knowledge Domain) — gehört nicht in KXF-Exports; Consumer liest `subjectCode`
- `CMP` (Component) — SSF-internes Konzept; aus KXF-Exports entfernen, in SSF-Modul-Dateien verschieben

---

## Kanonisches Feld-Schema nach Alignment

```json
{
  "id": "CHM-L1-000001",
  "legacyId": "LRN:SSF:CHE-1101",
  "subjectCode": "CHM",
  "title": "Warum reagieren manche Stoffe miteinander?",
  "entryQuestion": "Warum reagieren manche Stoffe miteinander?",
  "prerequisites": [],
  "complexity": "L1",
  "noxiaGrants": [
    { "key": "MISSION:LAB-ALPHA", "condition": "completed" }
  ],
  "status": "active"
}
```

- [ ] `subject_code` → `subjectCode`
- [ ] `depth_level` → `depthLevel`
- [ ] `noxiaKeys` / `unlocks` / `keys` → `noxiaGrants`
- [ ] `KD`-Referenzen aus KXF-Exports entfernen

- [ ] `legacyId` für alle umbenannten Module setzen
- [ ] System-Registry: `TEC`, `GEO`, `CHM` ergänzen
